# Ultima III 中文化 — 單色原始線條模式 評估

> 評估日期:2026-06-04
> 評估人:Claude(協助 L.CY / anr2)
> 結論:**可行且划算**。原版內建多平台 tileset 切換機制,單色資產已在庫,缺口集中在一個 stub。屬「打通既有機制 + 接選項 UI」,非新渲染管線。

---

## 0. 核心結論

LairWare 原版**內建多平台 tileset 切換**(`U3PrefTileSet` 偏好),且 `assets/graphics/` 已含多套**單色資產**(Apple II Mono、Macintosh B&W)。但 SDL 移植層的偏好讀取函式目前是 stub、永遠回 NULL,導致 tileset 永遠鎖死 `Standard`(彩色)。

**決定性證據**(`src/platform_sdl/plat_cf.c:229`):

```c
CFTypeRef CFPreferencesCopyAppValue(CFStringRef key, CFStringRef appID) {
    (void)key; (void)appID;
    return NULL;   // ← 永遠 NULL → GetGraphics 回退 "Standard"
}
```

`UltimaGraphics.c:319 GetGraphics()` 讀 `U3PrefTileSet` 拿到 NULL → `tilesNameRef = defaultTilesRef("Standard")`。**這就是要打通的那一個點。**

---

## A. 切在渲染層 / 資產層 / palette-filter 層?

**資產層已備妥、機制層已存在,真正要動的是「偏好讀取層 + 選項 UI」。Palette/filter 層不要碰。**

- 機制:`GetGraphics()` 用 `U3PrefTileSet`(預設 `"Standard"`)當前綴,從 graphics 目錄 prefix 匹配載入 `<name>-Tiles / -Mini / -Mask / -Font / -UI`,失敗逐項回退 Standard。
- 「單色模式」= 讓偏好回傳 `"Apple II Mono"`,其餘自動。
- **明確不建議 palette/filter 路線**(把 Standard 彩圖即時轉灰階/二值化):需新寫 shader、會糊掉線條、且本專案已有「真・原始單色資產」。濾鏡反而更難更醜。**用現成單色 tileset 才是真正的原始線條風格。**

## B. 能否保留單色線條風格並與繁中 UI/字型共存?

**能,耦合很乾淨。**

1. **中文 TTF 完全獨立**:中文走 `U3_DrawUTF8 → qd_text.c TTF_OpenFont(Noto CJK)`;tileset 的 `-Font` 載入到獨立的 `textPort`(ASCII 點陣字)。切 tileset 只換 ASCII 點陣字,**中文 TTF 不受影響**。
2. **選單中文按鈕正交**:9 個主選單 / Organize 按鈕用 `Buttons.png`(qd_text.c label_reconstruct),**不是 tileset `-UI`**。切 tileset 完全不動選單中文按鈕。
3. **唯一風險(POC 看截圖)**:遊戲內少量 ASCII 文字(HP/座標數字等)走 textPort 點陣字 → 切 Mono 後變像素風,與中文 Noto 並存可能「中文圓潤 + 數字像素」混搭。retro remake 常見、可接受,但需截圖確認觀感。

## C. 需要哪些檔案 / 函式?

| 檔案 / 函式 | 動作 | 說明 |
|---|---|---|
| `plat_cf.c:229 CFPreferencesCopyAppValue` | **解 stub** | 核心:對 `TileSet` 鍵回傳字串(shim 下 `CFStringRef`=`const char*`,回 strdup) |
| `cf_bridge.c`(偏好) | **加字串偏好** | 目前只支援「鍵→int」;需加字串值存 u3prefs.txt |
| `cf_bridge.c:87 pref_persist` 白名單 | 加 `"TileSet"` | 否則設定不持久化 |
| `plat_ui.c GameOptionsDialog` | 加一項 | 「畫面風格 [標準]/[單色]」循環,寫偏好 |
| `UltimaGraphics.c GetGraphics()` | **不改** | 機制現成;設定後重新呼叫即重載 |
| `plat_globals.c:41 U3PrefTileSet` | **不改** | symbol 已定義接上 |

`GetGraphics()` 有 3 個執行期呼叫點(UltimaNew×2、UltimaMain×1)→ **即時重載技術可行**。

## D. 選項開關與持久化設計

- **偏好鍵**:沿用原版 `TileSet`(字串:`"Standard"` / `"Apple II Mono"` / `"Macintosh B&W"`)。
- **持久化**:加白名單 → 存 u3prefs.txt,與 GameSpeed/Sound 同檔同機制。
- **UI**:GameOptionsDialog 加一項。⚠️ **memory P17 陷阱**:選項對話加行會讓 `ty` 累加在末行卡死 → 必須「逐行各自 `UPrintWin` 於明確 `wy`」,不可用含 `\n` 長字串。
- **生效方式**:首選「改後呼叫 GetGraphics 即時重載」;保底「提示重啟生效」。
- **預設**:`Standard`(不影響既有玩家)。

## E. Linux / Windows / AppImage 打包風險(低)

- **資產帶入**:`package_windows.sh:67 cp -r "$REPO/assets"` → **全帶**(含單色資產)。需同樣確認 `package_appimage.sh` 全帶 graphics。
- **檔名空格 / `&`**:`Apple II Mono-Tiles.png`、`Macintosh B&W-*`。GetGraphics 走 CFArray 目錄列舉 + prefix 比對,**不經 shell,空格 / & 安全**。打包 `zip -r` / `cp -r` 全目錄也安全。
- **無新依賴**:不是 shader 濾鏡,SDL_image 已能讀 PNG/GIF。**打包風險低。**

## F. 最小可行 POC + 測試 / 截圖驗收

**POC 目標**:先證「Mono tileset 在 SDL 移植版跑得通且與中文共存」,**不碰選項 UI / 持久化**。

- **最小改動**:`CFPreferencesCopyAppValue` 對 `TileSet` 改讀 env `U3_TILESET`(env-gated,無 env 維持 Standard,不影響正常遊玩)。
- **首選資產**:**Apple II Mono**(-Tiles/-Font/-Mini/-UI 齊全;Mac B&W 缺 Font/Mini/Mask 會回退混風)。
- **測試**:設 `U3_TILESET="Apple II Mono"` 跑 `build_and_verify.sh` → `tests/shots`。
- **pass/fail 截圖驗收清單**:
  1. 世界地圖 tile 變單色線條
  2. 主選單中文按鈕仍正常(Buttons.png 不受影響)
  3. sidebar 中文職業名 / HP 正常
  4. 底部中文訊息正常
  5. 戰鬥 / 城鎮畫面單色 + 中文 OK
  6. 觀察 ASCII 數字(textPort)與中文混搭觀感是否可接受

## POC 結果(2026-06-04)

### 原生 tileset 路線:接通但渲染受阻 🟠

- ✅ **切換機制接通**:解開 `plat_cf.c:CFPreferencesCopyAppValue` 的 stub(env `U3_TILESET` 餵入),`GetGraphics()` 確實改載指定 tileset(不再鎖死 Standard)。
- ✅ **中文解耦、選單正交**:切到 Apple II Mono 後,中文 sidebar / 底部訊息正常、主選單中文按鈕(Buttons.png)不受影響 —— 驗證 B 點結論成立。
- ❌ **影像渲染失敗**:Apple II Mono(colortype=3 palette PNG,tile 32×32)在上游 `GetGraphicTiledFile` 渲染成青色條紋。根因:`finalRect` = 768×1024(= Standard 尺寸),Standard 走 `straightDump` 等尺寸直貼(正常);非 Standard 尺寸走逐 tile `SDL_BlitScaled` 縮放路徑,palette PNG 經 GWorld/tile 佈局鏈失真。**把資產 nearest 預放大到 768×1024 仍失敗** → 非單純縮放,屬上游影像管線深層 bug,列為後續 backlog。

### 濾鏡路線:成功交付 ✅

改採 **renderer 層顏色濾鏡**(`main.c:U3_PlatPresent` 出口逐像素套色,不依賴 tileset):

- **四模式**:彩色(原樣)/ 綠磷光(Apple II 單色屏)/ 琥珀(amber CRT)/ 灰階。整數亮度運算 `y = (77R+150G+29B)>>8` 後重新上色。
- **動態切換**:`F2` 熱鍵(plat_event)/ 腳本 `V` 指令循環,即時生效,持久化 `ColorMode` 偏好(cf_bridge 白名單);登記於 F1 指令表。
- **與中文共存**:整屏統一轉換,中文一併變單色 → 風格比 tileset 混搭更一致。
- **驗收**:`tests/scripts/colormode.txt` 主選單循環四模式截圖確認(`docs/screenshots/colormode-*.png`);綠磷光世界地圖實機驗證 sidebar 中文一併單色。截圖反映濾鏡(present 截圖在 `gColorMode!=0` 時改從 renderer 讀回)。

> 結論修正:評估 A 點原排除 palette 濾鏡(怕糊線條),但實測原生 tileset 管線受阻;改用**亮度保留的色相濾鏡**(非二值化)不糊線條,且綠磷光正是 Apple II 單色屏真實外觀,反而最契合「動態切換」需求。tileset 深修列後續選項。
